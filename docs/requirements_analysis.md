# 《西游记观音院》游戏需求分析文档

## 1. 项目概述

### 1.1 项目背景
本项目为软件项目开发课程实训项目，以Pygame为开发框架，实现一款以西游记"祸起观音院"为故事背景的2D RPG游戏。玩家扮演孙悟空，需要在村庄中与NPC交互获取线索和属性加成，最终前往寺庙挑战牛怪。

### 1.2 项目目标
- 实现一个可操作的RPG游戏原型
- 掌握Pygame模块的使用
- 运用面向对象编程思想
- 熟悉软件开发流程

### 1.3 开发周期
| 阶段 | 时长 | 内容 |
|------|------|------|
| 集中学习 | 6天 | 学习Pygame基础知识 |
| 分散实现 | 6天 | 项目开发实现 |
| 答辩 | 1天 | 项目展示与答辩 |

## 7. 故事情节

### 7.1 背景
孙悟空保护唐僧西天取经，来到一座观音院前。需要找到土地公了解情况，得知寺院中有妖怪偷走了袈裟。与村民和唐僧修炼提升实力后，前往寺庙挑战牛怪。

### 7.2 角色设定
| 角色 | 位置 | 功能 | 素材 |
|------|------|------|------|
| 孙悟空(探索) | 村庄/寺庙 | 玩家控制主角，WASD移动 | swk2 (128帧PNG) |
| 孙悟空(战斗) | 战斗场景 | 战斗版玩家，WASD+鼠标攻击 | swk (16帧TGA) |
| 唐僧 | 村庄 | 剧情角色，交互提升攻击力 | 代码绘制简笔形象 |
| 村民 (elder1~4) | 村庄 | 交互提升血量 | elder目录 (TGA) |
| 土地公 | 村庄 | 剧情NPC，触发场景切换+失败鼓励 | god目录 (40帧TGA) |
| 牛怪 | 战斗场景 | 战斗敌人，自动追击 | cattle目录 (TGA) |

## 8. 详细设计规格

### 8.1 实时战斗系统设计

#### 战斗流程
```
进入战斗场景 → 加载地图 → 创建玩家(带属性) → 生成第一波敌人
    ↓
WASD移动 + 鼠标左键攻击 → 敌人追击AI → 伤害判定
    ↓
击败敌人 → 波次切换 → 下一波敌人
    ↓
全部击败/玩家死亡 → 战斗结束 → 结算画面
```

#### 战斗状态
| 状态 | 说明 | 转换条件 |
|------|------|----------|
| 战斗中 | 正在战斗 | 进入战斗场景 |
| 波次间隔 | 等待下一波 | 击败当前波敌人 |
| 胜利 | 战斗胜利 | 击败全部3波敌人 |
| 失败 | 战斗失败 | 玩家HP≤0 |

#### 攻击系统
| 要素 | 说明 |
|------|------|
| 攻击方式 | 鼠标左键点击 |
| 攻击范围 | 基于方向的挥砍范围（前方60px×80px） |
| 攻击冷却 | 无冷却，可连续快速点击 |
| 攻击锁定 | 攻击动画期间（20帧）玩家无法移动 |
| 伤害计算 | 玩家攻击力直接扣减敌人HP |

#### 敌人AI
| 要素 | 说明 |
|------|------|
| 追击速度 | 1.5像素/帧（比玩家慢） |
| 仇恨范围 | 300像素（进入范围开始追击） |
| 攻击范围 | 40像素（进入范围开始攻击） |
| 攻击冷却 | 60帧（约1.5秒） |
| 攻击方式 | 播放攻击动画，动画结束时造成伤害 |

### 8.2 玩家属性系统设计

#### 属性持久化
```
PlayerStats (单例)
├── hp: 当前血量
├── attack_power: 当前攻击力
├── elder_bonus_count: 村民加成次数
├── tang_bonus_count: 唐僧加成次数
├── total_enemies_defeated: 击败敌人总数
├── last_battle_won: 上次战斗结果
└── has_been_to_battle: 是否曾进入战斗
```

#### 属性传递流程
```
main.py创建PlayerStats → 传递给VillageScene/TempleScene/BattleScene
    ↓
VillageScene: 与Elder/Tang交互 → PlayerStats.add_elder_hp()/add_tang_attack()
    ↓
BattleScene.on_enter(): 读取PlayerStats属性 → 设置BattlePlayer.hp/attack_power
    ↓
BattleScene结束: 更新PlayerStats.total_enemies_defeated/last_battle_won
    ↓
SettlementScene: 读取PlayerStats显示结算信息
```

### 8.3 音效系统设计

#### 音乐配置
| 场景 | 背景音乐 | 循环播放 | 切换方式 |
|------|----------|----------|----------|
| 村庄 | bgm.mp3 | 是 | 渐弱→加载 |
| 寺庙 | bgm.mp3 | 是 | 渐弱→加载(强制重播) |
| 战斗 | fight.mp3 | 是 | 渐弱→加载 |

#### 渐弱切歌机制
```
当前音乐播放中 → 触发切换
    ↓
渐弱阶段 (30帧/约0.75秒): 音量从1.0线性降到0.0
    ↓
加载新音乐 → 播放 (音量恢复1.0)
```

### 8.4 受击效果设计

#### RGB通道变色
```
受击触发 → hit_timer = 10帧
    ↓
每帧检查: hit_timer % 4 < 2 时闪烁
    ↓
闪烁时: 降低绿色通道×0.5 + 降低蓝色通道×0.5 → 图片偏红
    ↓
10帧后恢复正常
```

### 8.5 渐变效果设计

#### FadeScene状态机
```
状态转换：
In (渐入) → Normal (正常显示) → Out (渐出) → Over (结束)
```

#### Alpha值变化
| 状态 | Alpha变化 | 步长 | 说明 |
|------|-----------|------|------|
| In | 0→255 | +20/帧 | 从透明到不透明 |
| Normal | 255 | 不变 | 正常显示 |
| Out | 255→0 | -20/帧 | 从不透明到透明 |
| Over | 0 | 不变 | 渐变结束 |

### 8.6 碰撞系统设计

碰撞检测采用两层设计，直接在各场景中实现：

#### 物理碰撞（停步检测）
```python
# 在场景的update方法中，将NPC/怪物碰撞框作为障碍物传入玩家更新
npc_rects = [npc.get_rect() for npc in self.npcs if npc is not self.current_npc]
self.player.update(keys, self.obstacles + npc_rects)
```

#### 交互检测（inflate扩大范围）
```python
# NPC交互检测 - inflate(20,20)扩大检测范围
player_rect = self.player.get_col_rect()
for npc in self.npcs:
    npc_rect = npc.get_rect().inflate(20, 20)
    if player_rect.colliderect(npc_rect):
        self.nearby_npc = npc
        break

# 怪物交互检测 - inflate(40,40)扩大检测范围
monster_rect = self.monster.get_rect().inflate(40, 40)
self.nearby_monster = player_rect.colliderect(monster_rect)
```

#### 脚部碰撞框
```python
# 碰撞框设为素材50%宽、35%高，底部对齐
self.col_w = int(self.width * 0.5)
self.col_h = int(self.height * 0.35)
self.col_offset_x = (self.width - self.col_w) // 2
self.col_offset_y = self.height - self.col_h - 2
```
